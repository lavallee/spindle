# clarify

Use when the request is broad, ambiguous, or missing acceptance criteria.

1. Identify the user's desired outcome in one sentence.
2. List the constraints, unknowns, and likely failure modes.
3. Ask only the questions that materially change implementation.
4. Produce a short "ready to design" brief with success criteria.

NEVER invent unstated requirements. Mark assumptions explicitly.
